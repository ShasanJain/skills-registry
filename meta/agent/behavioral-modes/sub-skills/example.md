# Behavioral Examples: Applying Cognitive Modes to Real Scenarios

Behavioral Examples provide concrete reference models demonstrating how the engine dynamically shifts cognitive gears in response to complex operational scenarios. By studying these transitions, agents learn to recognize subtle triggers and apply the exact operational mode (e.g., shifting from high-speed implementation to meticulous debugging) required for optimal execution without explicit human prompting.

## ⚙️ Scenario Recognition SOP

Follow this step-by-step procedure to apply cognitive modes appropriately:

- **Step 1: Scenario A - Green-Field Scaffolding**: When tasked with "Create a new dashboard app", immediately shift to **Explore & Brainstorm Mode**. Do not write components until the routing layer and design tokens are mapped.
- **Step 2: Scenario B - Feature Addition**: When tasked with "Add a billing page to the existing layout", shift to **Implement Mode**. Move fast, write clean modular components, and integrate with existing API endpoints directly.
- **Step 3: Scenario C - Stack Trace Encounters**: When a terminal command returns a complex error or stack trace, immediately halt forward momentum and shift to **Debug Mode**. Trace the failure back to its source before guessing fixes.
- **Step 4: Scenario D - Pre-Commit Audits**: When tasked with "Check if this is ready to push", shift to **Review Mode**. Activate maximum scrutiny for edge cases, performance leaks, and unauthorized data access.
- **Step 5: Scenario E - Complex Topic Explanations**: When the user asks "How does OAuth2 PKCE work?", shift to **Teach Mode**. Use structured Markdown, visual diagrams, and clear sequential explanations instead of terse code blocks.
- **Step 6: Synchronize and Evaluate Results**: After resolving the specific scenario, evaluate if the chosen mode successfully completed the objective. Log the transition in telemetry to reinforce correct behavioral mapping.
